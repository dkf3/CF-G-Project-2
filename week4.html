Week Four

GitHub and Version Control

In Week 4, we were introduced to GitHub and version control.

Version Control:

Version control is a tool which allows you to save the code you are working on frequently, but keeps a record of the poast versions.

This is very helpful for development, as you can go back and forth between versions of your code. But this is also helpful for group projects.

Within a collaborative code, version control allows you to experiment with new code without requiring that you save it to the master document, so you can try different things out before commiting your new additions to the larger project.

You can make both short and long term undos with version control

GitHub:

GitHub is a server where you can host websites and other coding projects.

GitHub has many detailed functions which help make coding easy for developers, such as version control. Within GitHub, one creates a repository on which all of your code is stored.

You can not only create repositories, but you can share repositories and clone the repositories you have access to.

To create code in your repository you can use GitHub or programs such as Atom. Then to save code, you must commit your document to a branch within your repository.

You can commit your code to the main, 'master' branch, or if you want you can create separate branches which you can later link or add to your master branch.

Once you are happy with your repository, you can make it public and publish it on GitHub so that other users can view it. You can also use GitHub to host your website.
